1. Постановка модели

Предлагается рассматривать воспринимаемую ценность продукта для конкретного потребителя (врача, клиники) как векторную величину в трёхмерном декартовом пространстве. Каждая ось соответствует одному из трёх базовых факторов принятия ценности:

Ось Фактор Смысловое содержание Примеры для SayWord
e⃗x Текущий контекст C Насколько внешняя среда благоприятна для внедрения: инфраструктура, регуляторика, позиция руководства, бюджетный цикл Клиника уже использует МИС; главный врач поддерживает автоматизацию; есть статья бюджета на IT
e⃗y Потребность N Острота проблемы, которую решает продукт: насколько велики потери без него Радиолог тратит 3+ часа на документацию; отделение перегружено; высокий риск ошибок из-за усталости
e⃗z Мотивация M Личная и организационная готовность к изменению: желание освоить новое, доверие к ИИ, карьерная выгода Врач молод и открыт к технологиям; руководство ставит KPI по скорости оборота исследований

Каждый фактор нормируется на интервале [0, 1], где 0 — полное отсутствие, 1 — максимальная выраженность. Длина вектора вдоль оси равна численному значению фактора.

2. Математическое определение результирующего вектора

Три компонентных вектора в базисе (e⃗x, e⃗y, e⃗z):

Определение компонент
V⃗C = C · e⃗x,    V⃗N = N · e⃗y,    V⃗M = M · e⃗z

Результирующий вектор ценности V⃗ есть их сумма:

Результирующий вектор
V⃗ = V⃗C + V⃗N + V⃗M  =  C · e⃗x + N · e⃗y + M · e⃗z

Его модуль (скалярная величина воспринимаемой ценности):

Модуль результирующего вектора
|V⃗| = √C² + N² + M²

Максимально возможное значение:  |V⃗|max = √3 ≈ 1,73  при C = N = M = 1

Направление вектора задаётся направляющими косинусами:

Направляющие косинусы
cos α = C|V⃗|,    cos β = N|V⃗|,    cos γ = M|V⃗|

Выполняется тождество:  cos²α + cos²β + cos²γ = 1

Направление вектора показывает, за счёт какого фактора формируется ценность. Два продукта с одинаковым модулем, но разным направлением требуют принципиально разных маркетинговых стратегий.

Обоснование оценок

Контекст (C = 0,75). Государственная программа цифровизации здравоохранения создаёт регуляторный попутный ветер. Большинство крупных клиник уже внедрили МИС и привыкли к интеграции внешних IT-решений. Сдерживающий фактор — бюджетные циклы в госсекторе и консерватизм части главных врачей. В коммерческих клиниках контекст ещё более благоприятен.

Потребность (N = 0,90). Наиболее сильный вектор. Задокументированные потери: 2–4 часа ежедневно на документацию при норме 8-часовой смены, высокий процент ошибок сторонности и пропущенных полей при работе под нагрузкой. Проблема универсальная, не зависит от размера клиники или региона.

Мотивация (M = 0,65). Молодые радиологи (до 40 лет) готовы к освоению ИИ-инструментов, старшее поколение проявляет скептицизм. Ключевой мотиватор — возможность уйти домой вовремя и снять тревогу об ошибках. Риск: часть врачей воспринимает автоматизацию как угрозу профессиональному статусу.

Вычисление результирующего вектора

Расчёт
|V⃗| = √(0,75)² + (0,90)² + (0,65)² = √0,5625 + 0,8100 + 0,4225 = √1,795 ≈ 1,34
Результирующий вектор ценности
|V| = 1,34
Из максимально возможных 1,73 — продукт находится в зоне высокой рыночной готовности. Соотношение к максимуму: 77%. Доминирующий фактор — потребность (ось Y, cos β = 0,67).
Направляющие косинусы — доля каждого фактора в результирующей ценности
cos α = 0,751,34 ≈ 0,56 (Контекст)

4. Анализ чувствительности: «что если»

Модель позволяет предсказать, как изменится результирующая ценность при целевых маркетинговых воздействиях.

Сценарий воздействия C N M |V| Изменение
Базовый (текущая оценка) 0,75 0,90 0,65 1,34 —
Вебинар + кейсы усиливают мотивацию до 0,85 0,75 0,90 0,85 1,47 +10%
Партнёрство с МИС улучшает контекст до 0,90 0,90 0,90 0,65 1,43 +7%
Оба воздействия одновременно 0,90 0,90 0,85 1,55 +16%
Консервативная клиника: контекст падает до 0,40 0,40 0,90 0,65 1,17 −13%
Клиника без перегрузки: потребность падает до 0,40 0,75 0,40 0,65 1,04 −22%
Ключевой вывод из анализа чувствительности: Снижение потребности (ось Y) даёт наибольшие потери ценности. Это означает, что маркетинговая коммуникация должна в первую очередь актуализировать проблему — показывать реальные потери времени и риски ошибок, — а затем уже предлагать решение.

5. Условие нулевой и максимальной ценности

Нулевая ценность. Даже при высоких значениях двух факторов, обнуление любого одного компонента существенно снижает модуль. Полный ноль достигается только при C = N = M = 0, что в реальности невозможно. Однако:

Практическое условие неприемлемо низкой ценности
|V⃗| < 0,5  ⟹  продукт не пройдёт порог принятия решения

Например: C=0,2,  N=0,3,  M=0,3  ⟹  |V⃗| = 0,47 — внедрение не состоится.

Максимальная ценность достигается при всех компонентах равных 1:

Максимум
|V⃗|max = √1² + 1² + 1² = √3 ≈ 1,73

SayWord сейчас на уровне 1,34 / 1,73 = 77% от теоретического максимума — сильная рыночная позиция.

6. Применение модели к сегментированию клиентской базы

Модель позволяет разбить потенциальных клиентов на сегменты по профилю вектора и выбрать для каждого оптимальный маркетинговый подход:

Сегмент Профиль (C, N, M) Стратегия
Идеальный клиент Высокие C, N, M (> 0,7 по каждой) Прямой выход, пилот без давления. Коммерческие многопрофильные клиники с молодым коллективом
Дрейфующий
(низкая M)		 C=0,8, N=0,9, M=0,4 Вебинары, кейсы, визиты «посмотреть в деле». Нужно дать опыт прикосновения к продукту, снять страх
Заблокированный
(низкий C)		 C=0,3, N=0,9, M=0,7 Работа с руководством клиники, а не с врачом. Письмо главному врачу, участие в конкурсах цифровизации, демо для ЛПР
Спящий
(низкая N)		 C=0,8, N=0,3, M=0,6 Нецелевой сегмент сейчас. Клиника с малым объёмом исследований. Вернуться при росте нагрузки

Выводы и рекомендации

  • Результирующий вектор ценности SayWord составляет |V| ≈ 1,34 из 1,73 — продукт находится в зоне высокой рыночной готовности.
  • Наиболее сильный компонент — потребность (N = 0,90). Маркетинг должен начинаться с демонстрации проблемы, а не решения.
  • Наиболее слабый компонент — мотивация (M = 0,65). Именно здесь сосредоточен главный рычаг роста: вебинары, live-демо, референции коллег дадут прибавку до +10–16% к результирующей ценности.
  • Сегмент «Заблокированный» требует отдельного B2B-трека — работы с администрацией клиник, а не с конечными пользователями.
  • Модель рекомендуется использовать как квалификационную матрицу в отделе продаж: перед выходом на переговоры оценивать C, N, M по чеклисту и прогнозировать вероятность сделки через |V|.
